El problema

Los equipos usan IA sin instrucciones de arquitectura

Cada sesión de IA empieza de cero. El asistente no sabe cómo trabaja tu equipo, qué decisiones se tomaron ayer, ni qué patrones han emergido en el código.

🧠

Memoria de pez

Cada sesión de AI empieza sin contexto. El asistente repite errores ya resueltos, propone soluciones que ya se descartaron, y no conoce las convenciones del equipo.

📋

Decisiones que se pierden

¿Por qué se usó SQLite en vez de Postgres? ¿Por qué no se usa Redux? Ese conocimiento vive en cabezas y Slack threads que nadie volverá a leer.

🔄

Inconsistencia entre sesiones

Un dev usa el AI de una forma, otro de otra. Sin guías compartidas, el código generado es inconsistente y difícil de revisar.

💸

Tokens quemados en contexto

Pegar el contexto completo en cada prompt es costoso. Sin un sistema de recuperación eficiente, los equipos o pagan demasiado o trabajan sin contexto.

Cómo funciona

Memoria persistente que se activa sola

Korva se instala en minutos. Funciona en segundo plano mientras el equipo desarrolla — sin cambiar el flujo de trabajo.

1

Instala korva-vault

Un binario Go. Se levanta como servidor MCP en localhost:7437. Sin Docker, sin dependencias externas. SQLite embebido.

2

Conecta tu editor

Manifiestos listos para Claude Code, Cursor, Windsurf, Copilot, Gemini y más. La IA descubre el vault automáticamente en cada sesión.

3

La IA captura contexto

Cada decisión, patrón y bug resuelto queda guardado con vault_save. El dedup semántico evita duplicados (Jaccard 0.70).

4

El equipo comparte skills

Los Skills de equipo se inyectan automáticamente en el contexto de cada sesión. Cero comandos manuales — el conocimiento del equipo aparece solo.

Funcionalidades

Todo lo que necesita un equipo profesional

Korva es más que un sistema de notas. Es infraestructura de IA para equipos que toman en serio la calidad del software.

🧩

Smart Skill Auto-Loader

En cada sesión, Korva detecta el proyecto activo y el prompt del desarrollador. Evalúa las skills del equipo por 4 señales (archivo 45%, keywords 30%, proyecto 15%, tags 10%) e inyecta las más relevantes automáticamente.

Teams
🔍

Dedup semántico

Antes de guardar cualquier observación, Korva calcula similitud Jaccard con el corpus existente. Si hay un registro ≥70% similar, lo sugiere en vez de crear un duplicado. La base de conocimiento se mantiene limpia sin esfuerzo manual.

Community
⚔️

Detector de conflictos

19 pares de antónimos arquitectónicos (sync/async, monolito/microservicio, SQL/NoSQL…). Cuando una decisión nueva contradice una existente, el agente recibe una alerta con la decisión conflictuante para que el equipo la resuelva conscientemente.

Community
🏥

Code Health Score

Puntuación compuesta A-F por proyecto. Analiza la distribución de tipos de observaciones, tendencias en el tiempo, y genera un informe de salud del código. Sin integración con CI — funciona directamente sobre el historial de decisiones del equipo.

Business
📈

Pattern Mining

Detecta patrones implícitos emergentes analizando las observaciones recientes del equipo. Identifica palabras y frases que aparecen con frecuencia anómala, sugiriendo qué convenciones deberían formalizarse como skills.

Business
🔐

Privacidad total

Todo funciona en tu infraestructura. El vault SQLite vive en ~/.korva/. El filtro de privacidad (internal/privacy) elimina automáticamente IPs, emails, tokens y keys antes de guardar cualquier contenido.

Community
🎛️

Beacon Dashboard

Panel de administración web (React 19 + Vite) para gestionar el equipo: crear skills, revisar el audit log, activar la licencia, ver la salud del código por proyecto, y administrar los scrolls privados del equipo.

Teams
🗜️

Compresión de output

vault_compress reduce el volumen de respuestas largas antes de guardarlas en memoria. Cuatro modos: off / lite / full / ultra. Configurado con KORVA_OUTPUT_MODE. Ahorra tokens en contextos token-intensivos.

Community
🏗️

SDD Workflow (Forge)

5 fases de Software Design Document directamente en el editor: Requisitos → Diseño → Implementación → QA → Deploy. El estado de fase se guarda por proyecto y es consultado automáticamente por la IA al inicio de cada sesión.

Arquitectura

Diseñado para producción enterprise

Monorepo Go. Binario único por componente. Sin CGO, sin dependencias externas en tiempo de ejecución. Se puede compilar, auditar y desplegar en cualquier infraestructura.

korva/ ← monorepo Go (go.work)
├── vault/ ← servidor de memoria principal
│ ├── korva-vault (binario) ← MCP stdio + REST API :7437
│ ├── internal/store/ ← SQLite + full-text search
│ ├── internal/mcp/ ← 23 MCP tools (JSON-RPC 2.0)
│ └── internal/api/ ← REST API + feature gates
├── cli/ ← CLI `korva` (Cobra + Bubbletea)
├── beacon/ ← dashboard React 19 + Vite 6
├── sentinel/ ← pre-commit hooks (Go)
├── lore/ ← scrolls de conocimiento (.md)
├── forge/ ← workflow SDD 5 fases
└── internal/ ← paquetes compartidos
├── license/ ← JWS RS256 offline validation
├── privacy/ ← PII filter (IPs, emails, tokens)
├── hive/ ← sync encriptado (Enterprise)
└── config/ ← platform paths, retention, webhooks

Stack tecnológico

Backend

Go 1.22+, SQLite (modernc — sin CGO), ULID monotónico, JSON-RPC 2.0, JWT/JWS RS256

Frontend (Beacon)

React 19, Vite 6, TypeScript. Embebido en el binario del vault — sin servidor web adicional.

Protocolo MCP

Model Context Protocol sobre stdio (JSON-RPC 2.0). Compatible con cualquier AI editor que soporte MCP.

Seguridad

PII filter automático, admin.key (0600), license JWS offline, rate limiter 120 req/min, audit log.

FAQ

Preguntas frecuentes

¿Los datos del equipo salen de nuestra infraestructura?

No. El vault SQLite vive en la máquina del desarrollador o en tu servidor interno. Korva nunca sube observaciones, skills ni scrolls a ningún servidor externo. El único tráfico externo es el heartbeat de licencia: license_id + install_id + timestamp.

¿Qué pasa si perdemos conectividad al servidor de licencias?

La validación de licencia funciona 100% offline mediante una firma RS256 embebida en el binario. Hay un período de gracia de 7 días sin conexión antes de que las features enterprise se suspendan. Los datos siempre son accesibles.

¿Necesitamos Docker o un servidor adicional?

No. korva-vault es un binario Go de archivo único. Se levanta con korva-vault --mode both y sirve tanto el servidor MCP como el dashboard web en el mismo proceso. SQLite embebido — sin Postgres, sin Redis, sin nada.

¿Funciona con nuestro stack NestJS / Fastify / Nx?

Sí. Korva no depende del lenguaje del proyecto — guarda observaciones de cualquier proyecto. Los Team Profiles se pueden configurar específicamente para NestJS hexagonal + Fastify + Nx, con convenciones, naming rules y patrones propios del equipo.

¿Qué pasa con las features community si cancelo la licencia?

Todas las features community permanecen disponibles indefinidamente. Las features Teams/Business se suspenden. Todos los datos guardados (observaciones, scrolls, skills) siguen accesibles — no se elimina nada.

¿Cuánto tiempo toma la instalación para un equipo de 10 devs?

El vault server se instala en ~5 minutos. Conectar los editores (copiar el manifest del IDE) toma 2 minutos por desarrollador. La configuración de Team Profile y Skills iniciales toma 30-60 minutos de setup. En menos de 2 horas, todo el equipo está con memoria compartida activa.

¿Podemos auditar el código fuente?

Sí. Para clientes Enterprise se proporciona acceso al repositorio privado para auditoría de seguridad. El código está escrito en Go estándar, sin ofuscación. Los paquetes de licencia y validación son los únicos componentes no-MIT.

¿Tiene soporte para Windows y Linux además de macOS?

Sí. Korva compila y corre en macOS, Linux y Windows. Los paths de datos se resuelven correctamente en cada plataforma via PlatformPaths() (XDG en Linux, AppData en Windows). La CI corre tests en los tres OS.
